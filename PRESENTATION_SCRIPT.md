# ðŸ—ï¸ 5. TECHNICAL ARCHITECTURE

### â±ï¸ Time: 2 minutes

> **[Show architecture diagram if available, or describe verbally]**

---

_"Under the hood, Cognito Learning Hub is a production-grade, enterprise-ready system."_

## Microservices Architecture

_"We built 9 independent microservices:"_

| Service                  | Purpose                                        |
| ------------------------ | ---------------------------------------------- |
| **API Gateway**          | Central routing, rate limiting, authentication |
| **Auth Service**         | JWT tokens, Google OAuth, role management      |
| **Quiz Service**         | Quiz CRUD, AI generation, file parsing         |
| **Result Service**       | Analytics, leaderboards, performance tracking  |
| **Live Service**         | Real-time sessions, Socket.IO, duels           |
| **Social Service**       | Friends, chat, notifications                   |
| **Gamification Service** | XP, achievements, streaks, quests              |
| **Moderation Service**   | Content safety, reports                        |
| **Meeting Service**      | WebRTC video, MediaSoup SFU                    |

_"Each service can be scaled independently."_
_"If quiz traffic spikes during exams, we scale only the Quiz Service."_

## Technology Stack

_"Our choices are deliberate and production-proven:"_

- **Frontend**: React 18, Vite, TailwindCSS, Framer Motion
- **Backend**: Node.js 20, Express.js
- **Real-time**: Socket.IO with Redis pub/sub
- **Database**: MongoDB Atlas (managed, globally distributed)
- **Caching**: Redis Cloud (sub-millisecond response times)
- **AI**: Google Gemini API
- **Video**: MediaSoup SFU (Selective Forwarding Unit)
- **Job Queues**: Bull for background processing

## Production Infrastructure

_"This is not a hackathon prototype. It's deployed and running."_

- **Frontend**: Vercel (global CDN, automatic scaling)
- **Backend Services**: Render (auto-healing, zero-downtime deploys)
- **Video Service**: Azure VM (dedicated resources for WebRTC)
- **TURN Server**: Self-hosted for NAT traversal

## 6.1 Quiz Generation Engine

_"When a teacher inputs a topic:"_

1. **Prompt Engineering**: We craft structured prompts that ensure consistent output format
2. **Context Enhancement**: Topic is enriched with educational taxonomy (Bloom's levels)
3. **Generation**: Google Gemini produces questions with options and explanations
4. **Validation**: Output is validated for format, correctness, and quality
5. **Storage**: Quiz is saved with metadata for future analytics

_"The same engine handles document parsing:"_

- PDF text extraction using PDF.js
- Word documents via Mammoth.js
- Key concept identification
- Question generation from extracted content

## 6.2 AI Study Buddy

_"Our AI tutor understands educational context:"_

- Maintains conversation history for contextual responses
- Adjusts explanation complexity based on student level
- Provides study strategies, not just answers
- Suggests related topics for deeper learning
