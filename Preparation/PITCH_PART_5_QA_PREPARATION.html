üîß TECHNICAL QUESTIONS

TECHNICAL
Q: How do you handle AI hallucinations in quiz generation?

A: We implement a three-layer validation system:

  • Layer 1 - Prompt Engineering: Strict output format specification in prompts
  • Layer 2 - Post-Processing: Validation that answers exist in options (prevent incorrect MCQs)
  • Layer 3 - Teacher Review: Teachers review and can edit/delete questions before publishing
  • Layer 4 - User Reporting: Students can report incorrect questions ‚Üí automatic flagging for review

Additionally, we maintain a "question quality score" based on student feedback and accuracy rates.

TECHNICAL
Q: How does your microservices architecture scale?

A: Each service scales independently based on load:

  • Quiz Service: High read volume ‚Üí multiple instances + Redis cache + CDN
  • Live Service: WebSocket intensive ‚Üí 5-10 instances with Socket.IO Redis adapter
  • Auth Service: Low load ‚Üí 2 instances sufficient
  • Result Service: High write volume ‚Üí MongoDB write replicas + Bull queues

API Gateway load balances across all instances. Database uses MongoDB sharding for horizontal scaling. This allows us to scale from 10K to 100K concurrent users by simply deploying more instances.

TECHNICAL
Q: What about data privacy and GDPR compliance?

A: We implement comprehensive privacy measures:

  • Encryption: AES-256 at rest, TLS 1.3 in transit
  • Right to Deletion: Account deletion triggers cascade delete of all user data
  • Data Minimization: Collect only essential data (no tracking pixels, minimal analytics)
  • Consent Management: Explicit opt-in for non-essential data collection
  • Auditing: Regular security audits and GDPR compliance checks
  • Data Portability: Export user data in standard formats (JSON, CSV)

We're ready for GDPR certification and working toward compliance with India's DPDP Act.

TECHNICAL
Q: How do you prevent cheating in multiplayer quizzes?

A: We use multiple anti-cheating mechanisms:

  • Server-Side Validation: All answers validated on backend (can't fake via client)
  • Time Windows: Enforce strict answer deadlines (no delayed submissions)
  • Question Shuffling: Each student gets questions in random order with randomized options
  • Pattern Detection: ML-based analysis of suspicious behavior (same answers across users)
  • Rate Limiting: Prevent rapid submissions
  • Moderator Review: Flagged games can be reviewed by moderators
TECHNICAL
Q: What's your AI API cost structure and is it sustainable?

A: Gemini API costs approximately $0.001 per quiz generation. Math:

  • Monthly Cost: 50,000 generations √ó $0.001 = $50/month
  • Monthly Revenue (Pro tier): 5% conversion √ó 50,000 users √ó $9.99 = $25K+
  • Gross Margin on AI: 99%+

As we scale, we'll optimize by caching common questions, using cheaper models for simple topics, and potentially training proprietary models. The free tier subsidy comes from institutional and premium tier revenue.

TECHNICAL
Q: How is real-time synchronization handled across services?

A: We use a combination of technologies:

  • Socket.IO: WebSocket connections for real-time events (1000s concurrent connections)
  • Redis Pub/Sub: Cross-service message broadcasting (service A publishes ‚Üí service B listens)
  • Event Streaming: Bull queues for guaranteed message delivery
  • Connection Rooms: Socket.IO rooms for efficient broadcasting (only relevant clients receive updates)

Example: Student submits answer ‚Üí Quiz Service validates ‚Üí publishes to Redis ‚Üí Live Service broadcasts to all ‚Üí Leaderboard updates in real-time for all viewers.

BUSINESS
Q: What's your competitive advantage over Kahoot/Quizizz?

A: Four key differentiators:

  • AI-First: They require manual quiz creation; we generate in 30 seconds
  • Comprehensive: We have quiz + tutor + video + social; they're single-feature
  • Deeper Gamification: 50+ levels vs their basic point systems
  • Voice Mode: Unique accessibility feature they don't have
  • Free Forever: Their premium model requires payment; we're free for students

Kahoot is a "game show," Quizizz is "homework delivery," Google Forms is "survey tool." We're a "learning ecosystem." Different market positions.

BUSINESS
Q: How do you acquire users in a crowded market?

A: Multi-channel growth strategy:

  • Viral Loop: Share quizzes ‚Üí invite friends (network effects)
  • Content Marketing: Free public quiz library ‚Üí SEO traffic
  • Institution Partnerships: Pilot programs with schools (B2B channel)
  • Community Building: Discord, Reddit, social media engagement
  • Referral Program: Both referrer and referred get Pro trial
  • Influencer Collaborations: Partner with student YouTubers/TikTokers

Our CAC (Customer Acquisition Cost) is nearly zero due to viral mechanics and referrals.

BUSINESS
Q: What's your path to profitability?

A: Clear profitability pathway:

  • Month 1-6: Focus on user acquisition (free tier)
  • Month 6-12: Conversion to Pro tier (target 5-8% conversion)
  • Month 12+: Institutional sales (high-margin B2B)

Revenue Model: Free tier (network) + Pro tier ($10/mo) + Institution tier ($100/mo)

Profitability Timeline: Cashflow positive by month 12-18 at 50K users. Breakeven on $2M funding in year 2.

Unlike most EdTech (high burn rates), we're designed for efficiency from day one.

PRODUCT
Q: Why should students use this vs just studying or using ChatGPT?

A: Three psychological hooks:

  • Gamification: Makes studying feel like gaming (XP, levels, achievements)
  • Social: Study with friends, compete on leaderboards, earn social status
  • Structure: Guided learning path vs free-form ChatGPT prompting
  • Feedback: Immediate assessment and performance analytics

ChatGPT is reactive ("explain X"). We're proactive ("here's a quiz on X, here's your weakness"). Plus gamification makes repeated practice fun.

PRODUCT
Q: How do you ensure content quality on the platform?

A: Multi-layer quality control:

  • AI Validation: Automated checks on generated quizzes
  • Teacher Review: Teachers must approve before publish
  • User Ratings: Public quizzes rated by community
  • Reporting System: Students report errors ‚Üí auto-flag for review
  • Moderators: Manual review of flagged content
  • Verified Badge: Educators with high-quality quizzes get "verified" status

Similar to how YouTube handles user-generated content - combination of automation + community + moderation.

PRODUCT
Q: What about mobile apps? Is web-only sustainable?

A: Mobile strategy:

  • Current: Fully responsive web app (works great on mobile browser)
  • Near-term (Q1 2026): Native iOS/Android apps using Flutter (single codebase)
  • Benefits of native: Better performance, offline mode, push notifications, app store discovery
  • Web will remain: Desktop teachers create quizzes on web; students take on mobile

Web-first approach allows fast iteration. Native apps come when we have user base to justify development cost.

PRODUCT
Q: How is this different from LMS platforms like Google Classroom?

A: Fundamentally different use cases:

  • Google Classroom: Class management (assignments, grades, announcements)
  • Cognito: Student engagement and learning (quizzes, gamification, social, AI tutor)

We're complementary - schools could use Google Classroom for admin + Cognito for learning. We're not an LMS, we're an engagement platform that can integrate with LMS.
