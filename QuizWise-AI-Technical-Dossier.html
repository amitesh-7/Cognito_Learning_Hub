1. Product Overview

  • Mission: AI-first learning platform enabling rapid quiz authoring, engaging gameplay, and collaborative communities.
  • Audience: Students, teachers, moderators, administrators.
  • Tech Pillars: React 18 SPA, Express 5 API, MongoDB Atlas, Google Gemini 2.5 Flash, JWT-secured REST, pdfExporter toolchain.
  • Deployment: Vercel SPA (`quizwise-ai.live`) backed by Render Node API (`quizwise-ai-server.onrender.com`).
  • Repository Layout: `quizwise-ai/` (frontend), `quizwise-ai-server/` (backend), `Steps/` (build log).

2. Role Matrix and Capabilities

Role Core Abilities Extended Privileges
Student Sign up/login (email or Google), browse quiz catalog, attempt quizzes, view personal dashboard, chat with AI tutor, participate in social hub, receive notifications, download PDFs. Initiate and respond to challenges, progress achievements/streaks, access leaderboards.
Teacher All Student features. Access quiz maker studio, edit/delete owned quizzes, view teacher analytics, export teacher/student/answer-key PDFs, interact with dedicated teacher communities.
Moderator All Student features. Moderation dashboard, resolve reports, edit/remove any quiz content, oversee community chats, enforce quality policies.
Admin All platform capabilities. Manage users and roles, access global analytics, create broadcasts, configure communities, audit health metrics.

Client-side guards (`ProtectedRoute`, `AdminRoute`, `ModeratorRoute`) and server-side middleware (`auth`, `adminMiddleware`, `moderatorMiddleware`) enforce zero-trust role validation.

3. Frontend Architecture

3.1 Application Shell

  • `App.jsx` wires routing, theme toggle, responsive menu, and wraps UI with `ToastProvider`.
  • Persistent theme preference via `useTheme` hook storing selection in localStorage.
  • Rich media (video hero, Lottie animations, particle overlays) served from `public/` assets.

3.2 State and Context

  • `AuthContext` manages JWT lifecycle, decoded `user` object, and isAuthenticated flag.
  • Local storage key `quizwise-token` validated against expiry to prevent stale sessions.

3.3 Routing and Protection

  • React Router v6 `Routes` with nested guards to redirect unauthorised access.
  • Navigation items adapt to user role (teacher dashboards, admin broadcast link, moderator tools).

3.4 UI Composition

  • Tailwind CSS utility system with `tailwind-merge` for conflict resolution.
  • Framer Motion animation variants (`fadeInUp`, `staggerContainer`) for polished transitions.
  • Custom UI primitives in `components/ui/` (Button, Input, Badge, Card, Toast) deliver consistent styling.
  • Gamified audio feedback triggered through `public/sounds` assets.

5. Feature Deep Dive

5.1 Quiz Authoring Studio

  • Topic Generator: `/api/generate-quiz-topic` enforces JSON responses, stores `Quiz` document with `createdBy` and metadata.
  • File Generator: Multer upload + `pdf-parse` extraction feeds Gemini for aligned questions.
  • Manual Creator: Rich form builder capturing question type, options, explanation, difficulty, points.
  • Enhanced Creator: Accepts prompts (learning objectives, desired tone) for richer AI output.
  • PDF Export: `QuizPDFGenerator` crafts HTML, `pdfExporter` renders to PDF or invokes print dialog.

5.2 Quiz Experience

  • Timed question flow with progress bar and immediate feedback.
  • Gamified bonuses, confetti, and sound cues reward streaks and perfect answers.
  • `POST /api/quizzes/report` captures content issues for moderator review.
  • Leaderboards via `GET /api/quizzes/:quizId/leaderboard` encourage replay.

5.3 Student Success Toolkit

  • Dashboard consumes `/api/results/my-results` for score trends and difficulty mix.
  • Achievement cards highlight progress toward next milestones.
  • Doubt Solver provides tutor-like answers with context retention.

5.4 Teacher Suite

  • `/api/quizzes/my-quizzes` returns teacher-created quizzes with derived stats and sorting.
  • Inline actions (preview, edit, delete, export) streamline management.
  • Teacher achievements motivate content creation through XP and badges.

5.5 Social Hub & Collaboration

  • Friend lifecycle: request (`POST /api/friends/request`), respond (`PUT /api/friends/respond/:id`), list (`GET /api/friends`).
  • Challenges stored with duel results, winner selection, expiry timers.
  • Chat endpoints handle message persistence, room membership, and read receipts.
  • Notifications API delivers friend/challenge/broadcast updates.

5.6 Moderation & Administration

  • Reports stored with priority and status, surfaced in moderator dashboard.
  • Admin broadcasts (`POST /api/broadcasts/create`) support audience filters and urgency levels.
  • Privileged deletes/edits allow rapid content correction by moderators/admins.

6. Analytics and Reporting

  • Student Dashboard: Line chart (scores over time), pie chart (difficulty mix), streak counter, achievements carousel.
  • Teacher Dashboard: Totals (quizzes, takes, unique students), average rating, sortable quiz list, trending indicators.
  • Admin Dashboard: Role distribution, active users, daily attempts, report backlog metrics.
  • Moderator Dashboard: Filters by status/priority, SLA countdown, assigned moderator metadata.
  • Achievement Dashboard: XP bar, level thresholds, rarity filters (`common` → `legendary`).

7. AI Services

  • Gemini 2.5 Flash client initialised once per server using `.env` `API_KEY`.
  • Prompt templates enforce JSON arrays, include teacher-provided metadata for context-rich quizzes.
  • `extractJson` helper guards against malformed responses.
  • Doubt Solver accepts conversation history and returns structured explanations with fallback messaging on failure.
  • AI outputs validated before persistence to prevent storage of partial/invalid data.

8. Backend Architecture

8.1 Core Stack

  • Express app with JSON parsing, CORS, and consistent error logging.
  • MongoDB connection with connection-state logging for deployment diagnostics.
  • Server listens on port 3001 (Render aligned); environment toggles adjust CORS.

8.2 Middleware

  • `auth` middleware validates `x-auth-token`, attaches `req.user`, handles expiration.
  • `adminMiddleware`, `moderatorMiddleware` ensure proper role-based access.
  • Multer disk storage isolates uploads under `/uploads` with deterministic filenames.

8.3 Services & Helpers

  • `extractJson` cleans AI responses for safe parsing.
  • `checkUserAchievements` updates `UserStats`, unlocks achievements, awards XP.
  • `QuizPDFGenerator` builds semantic HTML with metadata, question badges, teacher notes.
  • Leaderboards, challenge results, and notifications computed via Mongoose aggregation.

9. Data Models

  • User: identity (name, email), auth (password or Google ID), role, picture, status, activity timestamps.
  • Quiz: metadata (title, description, category, tags, visibility), question subdocs with type, options, difficulty, time limit, points, explanations, ratings.
  • Result: quiz + user references, score, percentage, pass flag, time metrics, streak snapshot, XP, question-level breakdown.
  • Achievement / UserAchievement / UserStats: Criteria, rarity, points, progress, badges, experience, level, favourite categories.
  • Report: question text, reason enum, priority, status, resolver metadata, timestamps, indexes for dashboards.
  • Social Suite: Friendship, Challenge, Message, ChatRoom, Notification, Broadcast capture social graph and communication history.

10. API Catalog

10.1 Authentication

  • `POST /api/auth/register` — Standard sign-up (role selection, hashed passwords).
  • `POST /api/auth/login` — Email/password login issuing JWT.
  • `POST /api/auth/google` — Verify Google ID token, provision account, return JWT.
  • `POST /api/auth/google/update-role` — Persist chosen role for Google accounts (requires JWT).

10.2 Quiz Management

  • `GET /api/quizzes` — Public catalog with creator names.
  • `GET /api/quizzes/:id` — Retrieve quiz detail.
  • `GET /api/quizzes/my-quizzes` — Teacher analytics (`sortBy` query).
  • `POST /api/generate-quiz-topic` — AI topic generation (auth).
  • `POST /api/quizzes` — Save manual quiz.
  • `PUT /api/quizzes/:id` — Update quiz (author/moderator/admin).
  • `DELETE /api/quizzes/:id` — Delete quiz (author/moderator/admin).
  • `POST /api/quizzes/enhanced` & `/api/quizzes/generate-enhanced` — Enhanced AI templates.
  • `POST /api/quizzes/:id/submit` — Persist result.
  • `GET /api/quizzes/:quizId/leaderboard` — Leaderboard data.
  • `POST /api/quizzes/generate-pdf` / `GET /api/quizzes/:id/pdf/:format` — Export HTML for PDFs.
  • `POST /api/quizzes/report` — File content report.

10.3 Results & Achievements

  • `GET /api/results/my-results` — Student dashboard dataset.
  • `GET /api/achievements/stats` — Combined stats, comparisons, recent unlocks.
  • `GET /api/achievements/catalog` — Achievement definitions (when enabled).

10.4 Social & Communication

  • `POST /api/friends/request`, `PUT /api/friends/respond/:id`, `GET /api/friends`.
  • `POST /api/challenges/create`, `GET /api/challenges`.
  • `GET /api/notifications`, `PUT /api/notifications/:id/read`.
  • `POST /api/chat/send`, `GET /api/chat/messages/:friendId`, `GET /api/chat/rooms`, `PUT /api/chat/read/:friendId`.
  • `POST /api/broadcasts/create`, `GET /api/broadcasts`.

10.5 AI & Tutor

  • `POST /api/doubt-solver` — AI tutor response.
  • `POST /api/quizzes/generate-from-file` — File-based quiz generation (with Multer upload).

10.6 Administration & Moderation

  • `GET /api/reports`, `PUT /api/reports/:id/status` — Report workflow.
  • `GET /api/admin/users`, `PUT /api/admin/users/:id/role` — User role management.

Refer to quizwise-ai-server/index.js for exact payload schemas, middleware guards, and supplementary endpoints.

11. Security and Compliance

  • JWT expiration enforced client and server side; tokens removed on failure.
  • Password hashing via bcrypt; schema makes password optional if Google ID stored.
  • Role-based guards on frontend and backend for defence in depth.
  • Environment-based CORS whitelist; credentials explicitly allowed when required.
  • Multer upload path constrained to `/uploads`, preventing arbitrary file writes.
  • Report workflow plus moderator tools uphold content quality and compliance.
  • Broadcast read receipts and timestamps supply audit trail.

12. DevOps and Deployment

  • Frontend deployed on Vercel with `vercel.json` rewrites; environment variables configure API base per stage.
  • Backend hosted on Render Node service running `npm start` with `.env` secrets.
  • Vite build generates static assets; server remains unbundled for quick cold starts.
  • Static assets (videos, animations) served via Vercel CDN.
  • `NODE_ENV` toggles CORS whitelists and other production behaviours.

13. Developer Experience

  • Scripts: `npm run dev`, `npm run build`, `npm run preview` (client); `npm start` (server).
  • Tailwind tooling, PostCSS autocomplete, lucide-react icon library speed UI development.
  • Reusable utilities (`lib/utils.js`, `lib/pdfExporter.js`) centralise animation and export logic.
  • `Steps/` folder documents chronological milestones: Tailwind setup, OAuth, dashboards, deployments.
  • Consistent component patterns reduce cognitive load while adding new features.

14. Observability and Roadmap

  • Current logging relies on console statements; upgrade path includes structured logging (Winston/Pino) and hosted log sinks.
  • Introduce automated tests (Vitest/Jest, React Testing Library, Supertest) to prevent regressions.
  • Adopt WebSockets (Socket.IO) for real-time chat and presence.
  • Centralise shared constants (roles, achievement thresholds) to avoid duplication.
  • Implement rate limiting for AI endpoints and add analytics instrumentation (Segment/Mixpanel).
